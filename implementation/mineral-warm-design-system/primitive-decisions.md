# Primitive Registry Equivalence Decisions

## Audit of `src/components/ui/`

The existing UI primitives in `src/components/ui/` appear to be generated by the `@react-native-reusables` CLI. They depend on `react-native-css-interop` and TailwindCSS utility classes.

### Decisions:
1. **Text**: Needs to map directly to `TYPOGRAPHY` roles (display, title, subtitle, body, highlight, label, caption) instead of passing raw font weights and sizes.
2. **Button**: Current variants (default, destructive, outline, secondary, ghost, link) must map to Mineral Warm semantics. Specifically:
   - Destructive uses tomato colors.
   - Outline uses `borderStrong` or `borderControl`.
   - Secondary maps to moss or auxiliary.
   - Link uses `text-link`.
3. **Card**: Needs to map to surface, surfaceElevated, and respect strict border radius and border colors.
4. **Badge**: Needs tone mapped to amber, tomato, moss, or neutral scales depending on semantic emphasis.
5. **Progress & Separator**: Use primary and `borderSubtle`.
6. **Input & Label**: Map to controlled entry states (focus using `borderFocus` which is blue, error using tomato).

## Dependency / Registry Additions
No new primitives need to be fetched via `npx @react-native-reusables/cli add`. The current set of primitives is sufficient to satisfy the immediate feature scope. The diffs will be limited to modifying the class names within the existing `src/components/ui/*.tsx` files to strictly use our defined semantic aliases from `tailwind.config.js` and `global.css`.
