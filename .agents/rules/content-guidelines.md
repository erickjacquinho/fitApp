# Content Guidelines

FitApp UI copy should be short, direct, and written in Brazilian Portuguese.

## Voice
- Use simple action-oriented language.
- Prefer short labels over explanatory text.
- Avoid playful or vague messages in operational flows.
- Do not explain the interface when the control label is enough.

## CTAs
- Start with a verb.
- Use one primary CTA per screen section.
- Destructive CTAs must be explicit: avoid generic text like `Confirmar`.

Examples:
- `Adicionar treino`
- `Salvar refeição`
- `Excluir registro`

## Labels
- Labels should be scannable and specific.
- Avoid long field labels.
- Use helper text only when the field needs clarification.

## Errors
Error messages should include the cause and the next action when possible.

Good:
- `Informe um peso válido.`
- `A refeição precisa de pelo menos um alimento.`

Avoid:
- `Erro`
- `Algo deu errado`

## Empty States
Empty states need:
- short title
- one sentence of context
- one clear CTA when action is possible

## Confirmations
- Destructive confirmations must name the affected item or action.
- Success messages should be brief and should not block repeated input.
